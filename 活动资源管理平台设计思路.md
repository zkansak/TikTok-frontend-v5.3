## 🎯 核心需求分析

### 用户原始需求
- **前端平台型架构**：展示架构能力

| 维度 | 当前实现 | 理想实现（参考CMS/广告平台） |

| **核心功能** | CRUD管理 | 资源配置 + 触达投放 |

**核心功能**：

## 🚀 改进方案：资源管理平台

### 方案A：完整的资源管理平台（推荐）

#### 1. 资源库管理
```
- 资源上传（图片、视频）
- 资源预览
- 资源分类（Banner、弹窗、视频等）
- 资源删除/编辑
```

#### 2. 活动配置
```
- 创建活动
- 选择资源（从资源库选择）
- 配置投放规则：
  - 投放位置（首页顶部、首页底部、全站等）
  - 投放时间（开始/结束时间）
  - 显示规则（首次访问、每次访问等）
  - 人群定向（可选）
```

#### 3. 触达效果
```
- 在首页实际展示配置的Banner/弹窗
- 支持关闭状态缓存（localStorage）
- 支持跳转链接
```

#### 4. 数据统计（可选）
```
- 展示次数
- 点击次数
- 点击率
```

## ❓ 需要确认的问题

1. **资源来源**：
   - 仅URL输入（当前）？
   - 还是需要资源上传功能？

2. **投放位置**：
   - 仅首页？

3. **显示规则**：
   - 首次访问显示？
   - 每次访问都显示？
   - 支持关闭后不再显示？

4. **优先级**：
   - 当前实现是否满足需求？
   - 是否需要更复杂的优先级规则？
